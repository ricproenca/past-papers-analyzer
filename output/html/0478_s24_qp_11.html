Q9b_i 8 marks AO2 Describe 2.2 Methods of Error Detection p.10▾

A company owner has installed a new network. Data is correct before it is transmitted across the network. The company owner decides to introduce an error detection system to check the data for errors after transmission. The error detection system uses an odd parity check and a positive automatic repeat query (ARQ).

Describe how the error detection system operates to check for errors.

Marking scheme

Any eight from:

  • The 1s are counted (in each byte)[1]
  • Each byte has a parity bit[1]
  • If the number of 1s are odd the parity bit is 0 (otherwise it is 1)[1]
  • (The first packet of) data is sent and a timer is started[1]
  • The receiving device counts the number of 1s (in each byte)[1]
  • If the number of 1s are odd/data meets odd parity an acknowledgement is sent to say the data is error free …[1]
  • … the sender then sends the next packet of data …[1]
  • … and the timer is restarted[1]
  • If the number of 1s is even an acknowledgement is not sent[1]
  • If no acknowledgement is received within a set timeframe/before timeout …[1]
  • … the data packet is resent[1]
